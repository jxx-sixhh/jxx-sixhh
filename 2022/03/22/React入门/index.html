JSX

  1. react介绍JSX

    JSX是JS的一个语法扩展，是React.createElement函数的语法糖

  2. react里面JSX直接绑定事件有啥缺点

    ⭐第一种：onClick = {this.yourEvent} 需要在构造函数中绑定this

     第二种：`onClick = {this.yourEvent.bind(this)}` //每次render,都会重新生成一个函数
 第三种：`onClick = {() => this.yourEvent()}` //每次render，都会执行一遍方法

  3. JSX的本质;

    实际上JSX是React.createElement(component,props,...children)函数的语法糖

  4. 为什么在JSX里循环生成DOM需要添加key

    在进行diff算法时，如果没有key的话，在列表数据渲染过程中，如果我后来又在前面又添加了一条数据，那么所有他后面的数据都要被修改。如果有key的话，react将根据key属性匹配原有树上的子元素以及最新树上的子元素，这样的话只需要把最新的那个元素插入到最前面位置即可。

  5. 浏览器怎么识别JSX的语法

    浏览器是无法识别JSX的，所以我们需要依靠babel或者ts来将JSX转换为JS。在react17之前，都是把他转换成react.createElement()函数，在react17后，它是自动从 React 的 package 中引入新的入口函数并调用。

  6. 浏览器不能识别JSX文件，需要怎么办？

    配置@bebel/preset-react,用于处理JSX类型的文件，将其转换为JS文件

  7. JSX怎么转成原生JS。JSX怎么用React.createElement创建?

    JSX与JS在线转换器

  8. 项目中为什么要用JSX语法

用户定义的组件必须以大写字母开头,如果使用了小写开头，那么React会错误的认为这个组件是一个原生dom标签。例如，<div /> 代表 HTML 的 div 标签，而 <Welcome /> 则代表一个组件，并且需在作用域内使用 Welcome。
