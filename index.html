String -> Number

parseInt(string,radix)

该函数解析一个字符串并返回指定基数的十进制整数

参数

string 必需
radix 可选，表示以几进制来读取字符串，取值范围是2-36。请注意radix的默认值并不是10，并且在使用parseInt()函数时请一定要传入radix参数。在后文中“一些注意的点”中将会解释为什么我们必须这么做。

返回值

解析后的十进制数
或者NaN

一些注意的点

截断解析

请一定指定radix

当radix是undefined、0、或者压根没有的时候，js会假定以下情况。
如果string以0x(x无论大小写)开头，那么radix会假定为16。
如果string以0开头，那么radix会假定为8/10。。ES5澄清了会假定为10，但是！并不是所有浏览器都支持。因此，在使用parseInt时，一定要指定一个radix

1. 双指针

ex1:合并两个有序数组


看到这道题你可能跟我一样首先想到的是先合并后排序，不过面试的时候要是敢这么写，面试官可能当场就说“好，今天的面试就到这里了”。后来或许我们可以想到再新建一个数组，利用双指针比较nums1和nums2头部的数字，取出比较小的存在新数组里。不过，这种方法还是可以继续优化。因为我们可以看到nums作为结果数组，其本身的长度是足够长的，所以我们可以将指针从右往左倒序遍历，取出比较大的放在nums1的后部。这样空间复杂度便可变为O(1)，因为我们没有占用额外空间

为什么js可以在浏览器中被执行？因为浏览器中都有js解析引擎。

什么是node.js?node.js是基于chrome v8引擎的js运行环境

为什么js能够被浏览器执行？因为各大浏览器中都内置的有js解析引擎，例如chrome浏览器使用的是v8引擎

为什么js可以操作dom和bom？因为每个浏览器都内置了dom，bom这样的API函数，因此js可以调用这些web api函数，再被js解析引擎解析，从而在浏览器中被执行

浏览器是js的前端运行环境，node.js是js的后端运行环境，node.js中无法调用dom和bom等浏览器内置api

第 1 章：Promise 的理解和使用

1.1. Promise 是什么?

1.1.1. 理解

  1. 抽象表达:

    1. Promise 是一门新的技术(ES6 规范)

    2. Promise 是 JS 中进行异步编程的新解决方案

    备注：旧方案是单纯使用回调函数

  2. 具体表达:

    1. 从语法上来说: Promise 是一个构造函数

    2. 从功能上来说: promise 对象用来封装一个异步操作并可以获取其成功/失败的结果值
